# 已故科学家候选池 V1：120 位历史科学人物名单

## 1. 文档目标

本文提供轻测试 / 画像系统可继续深入研究的第一版 `120 位已故科学家候选池`。

这份名单的用途不是直接当最终类型映射，而是作为后续工作的基础池：

1. 为每种画像寻找主锚点和备选锚点
2. 为类型命名提供历史人物参照
3. 为“科学人物参考”卡片建立可靠来源池

---

## 2. 本名单的使用原则

- 只收 **已故人物**
- 采用广义“科学人物”口径：
  - 含数学家
  - 含理论计算与信息科学人物
  - 含工程科学与系统科学人物
  - 含自然史与实验科学人物
- 优先考虑：
  - 公众相对可识别
  - 研究风格有叙事辨识度
  - 后续容易找到 2 个以上可靠来源
- 当前版本追求的是：
  - **覆盖度**
  - **候选池质量**
  - **后续可继续画像化**

说明：

- 这不是最终定稿名单
- 这不是最终类型映射表
- 这也不是“谁更伟大”的排序

## 4. 这 120 人为什么适合作为第一版候选池

这份名单不是按“学科地位”单一维度选的，而是综合考虑了五件事：

1. 是否是明确已故人物
2. 是否具有相对稳定的公众识别度
3. 是否有比较鲜明的研究风格叙事
4. 是否能为后续的画像系统提供不同气质样本
5. 是否能让后续类型收缩时保留足够冗余，不至于出现某些画像维度无可用人物

这也是为什么名单里会同时出现：

- 极端理论型人物
- 长期深挖型人物
- 实验与观察型人物
- 跨界整合型人物
- 工程系统型人物
- 公共表达能力较强的人物
- 偏沉潜、偏抽象、偏内向的人物

---

## 5. 当前不要急着做的事

基于这份名单，当前还不建议立刻做下面两件事：

1. 不建议直接给 120 人强行打完整人格分数
2. 不建议马上把 120 人一对一硬映射到某个固定类型

原因：

- 这一步更适合先做结构化定性画像
- 如果过早量化，很容易凭印象贴标签
- 如果过早定型，后面会限制命名系统和人物锚点选择空间

---

## 6. 更适合的下一步

基于这份 `120 位候选池 V1`，后续更稳的顺序是：

1. 先制定“锚点画像摘要”的统一模板
2. 给这 120 人各写一版简短画像摘要
3. 再做第一轮收缩，缩成 `20-30 位强锚点人物`
4. 最后再映射到 `4+1` 画像系统

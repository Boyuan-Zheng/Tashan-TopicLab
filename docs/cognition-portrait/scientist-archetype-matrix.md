# Scientist Archetype Matrix

## 1. 文档目标

本文把“科学家如何分类”正式收口成一套可执行骨架。

它回答的不是：

- 轻测试怎么写题
- 分享卡怎么命名
- 单个科学家 skill 怎么写

而是：

**如果我们最终要把科学家分成一套稳定、有限、可解释的画像类型，应该怎么分。**

---

## 2. 先说结论

当前最稳的方案是：

- 用 `4` 个主轴定义 `16` 个主原型
- 再用 `RCSS` 的 `I / D` 作为后缀轴
- 最终得到：
  - `16` 个主原型
  - `32` 个完整科学家画像类型

也就是说：

- `主原型` = `2^4 = 16`
- `完整类型` = `16 × 2 = 32`

这套结构是当前最适合 TopicLab 的折中：

1. 前四轴有相对稳的心理学骨架
2. 最后一轴保留你们自己的“科研风格感”
3. 类型数既不会太少，也不会爆炸

---

## 3. 主轴定义

## 3.1 四个主轴

主轴来自前面已经讨论过的四条高层人格轴：

1. `O`：开放性  
   `高 O` = 爱新问题、抽象跳跃、愿意跨界  
   `低 O` = 更偏稳定问题、经典框架、熟悉方法

2. `C`：尽责性  
   `高 C` = 结构、纪律、长期推进、完成度高  
   `低 C` = 更松、更试探、更依赖灵感或局部推进

3. `E`：外向 / 公共在场  
   `高 E` = 更愿意公开表达、解释、组织、带人、进入场域  
   `低 E` = 更沉潜、低调、边界强、少舞台感

4. `S`：情绪稳定性  
   `高 S` = 面对压力、争议、失败时更稳  
   `低 S` = 更容易被卷入焦虑、张力、羞耻、波动

说明：

- 这里故意用 `S` 表示 `Stability`，而不是直接写 `N`
- 对产品和分类表达来说，正向写成“稳定性”更直观

---

## 3.2 科研风格后缀轴

第五位不进主原型，而是作为后缀：

- `I = Integration`
- `D = Depth`

含义：

- `I`：更偏横向整合、跨概念拼装、抽象连通、系统视角
- `D`：更偏纵向深挖、长期打磨、单点突破、耐力推进

所以最终编码形式建议统一为：

`[O/C/E/S 主原型]-[I/D]`

例如：

- `OCES-I`
- `OCeS-D`

这里大小写只是示意，不要求最终产品真的把编码直接暴露给用户。

---

## 4. 为什么是 16，而不是 12

因为我们现在讨论的是**分类骨架本身**，不是产品展示层。

如果按四个二元主轴做主分类，那么：

- `2 × 2 × 2 × 2 = 16`

这是数学上和结构上都最清楚的主原型层。

后面可以在产品展示时压缩、合并或只重点运营其中一部分，
但分类系统本身应该先完整保留 `16`。

---

## 5. 16 个主原型

下面这 16 个名字是**工作名**，目标是：

- 比纯编码更可读
- 比“严肃学术标签”更有人味
- 但仍然保持结构可解释

后续对外展示时，这些名字还可以继续微调。

## 5.1 高 O / 高 C / 高 E / 高 S

### 1. `开路统筹者`

- 画像核心：既爱新问题，也有推进力，还能在公共场景里稳稳带人
- 典型气质：方向感强、组织力强、解释力强
- 候选科学家：
  - `Qian Xuesen`
  - `Grace Hopper`
  - `Leonardo da Vinci`

### 2. `开路张扬者`

- 画像核心：高开放、高行动、高在场，但稳定性略弱，更容易带戏剧张力
- 典型气质：能量强、影响大、冒险感更强
- 候选科学家：
  - `Richard Feynman`
  - `Nikola Tesla`

### 3. `开路稳行者`

- 画像核心：高开放、高行动、低在场但稳定，像低调的高压推进机器
- 典型气质：不爱舞台，但很能做成事
- 候选科学家：
  - `Marie Curie`
  - `Barbara McClintock`

### 4. `开路波动者`

- 画像核心：高开放、高行动、低在场、稳定性较低，常带强张力或孤绝感
- 典型气质：深刻、执拗、容易和环境发生摩擦
- 候选科学家：
  - `Lise Meitner`
  - `Ludwig Boltzmann`

## 5.2 高 O / 高 C / 低 E / 高 S

### 5. `深构建模者`

- 画像核心：高开放、高尽责、低在场、高稳定
- 典型气质：极强的建模和长期推进能力，不靠舞台感赢
- 候选科学家：
  - `Claude Shannon`
  - `Emmy Noether`
  - `John von Neumann`

### 6. `张力建模者`

- 画像核心：高开放、高尽责、低在场，但稳定性较低，容易在高压中形成强张力
- 典型气质：抽象、锋利、容易带痛感
- 候选科学家：
  - `Alan Turing`
  - `Kurt Godel`

### 7. `静稳开拓者`

- 画像核心：高开放、低尽责、高在场、高稳定
- 典型气质：有探索感，也有传播和带动能力，但推进风格更灵活
- 候选科学家：
  - `Carl Sagan` 不在池中，当前可先保留空位思路
  - `Buckminster Fuller`

### 8. `躁动开拓者`

- 画像核心：高开放、低尽责、高在场、稳定性较低
- 典型气质：灵感、冒险、跳跃、公共张力
- 候选科学家：
  - `Thomas Edison`
  - `Nikola Tesla` 也可能落这里，视后续细分

## 5.3 高 O / 低 C / 低 E / 高 S

### 9. `冷静游侠`

- 画像核心：高开放、低尽责、低在场、高稳定
- 典型气质：自由探索、低调、不太被制度束住
- 候选科学家：
  - `Paul Dirac`
  - `Srinivasa Ramanujan`

### 10. `孤绝游侠`

- 画像核心：高开放、低尽责、低在场、稳定性较低
- 典型气质：天才张力强、边界感重、难以被环境完全容纳
- 候选科学家：
  - `Georg Cantor`
  - `Ludwig Boltzmann`

## 5.4 低 O / 高 C / 高 E / 高 S

### 11. `秩序统筹者`

- 画像核心：更重结构、执行、组织和稳定，不靠高开放取胜
- 典型气质：平台型、制度型、组织型
- 候选科学家：
  - `Vannevar Bush`
  - `W. Edwards Deming`

### 12. `秩序施压者`

- 画像核心：高结构、高在场、稳定性较低
- 典型气质：能推、能压、能组织，但也更容易带控制感
- 候选科学家：
  - 当前池里暂不急着硬贴一个“完美人物”
  - 先保留为结构位，后续从工程/组织人物里细筛

### 13. `稳态工匠`

- 画像核心：低开放、高尽责、低在场、高稳定
- 典型气质：耐心、稳、可靠、长期磨一个东西
- 候选科学家：
  - `Gregor Mendel`
  - `Robert Koch`
  - `Rosalyn Yalow`

### 14. `受压工匠`

- 画像核心：低开放、高尽责、低在场、稳定性较低
- 典型气质：能熬，但也容易承受巨大内耗与张力
- 候选科学家：
  - `Rosalind Franklin`
  - `Inge Lehmann`

## 5.5 低 O / 低 C / 高 E / 高 S

### 15. `公众解释者`

- 画像核心：更强的是在场、解释、影响和带动，而不是极端抽象或极端纪律
- 典型气质：适合做桥梁、讲解、公共传播
- 候选科学家：
  - `Alexander Graham Bell`
  - `Guglielmo Marconi`

### 16. `公众冒险者`

- 画像核心：低结构、高在场、稳定性较低
- 典型气质：更像带着冲劲和赌性往前拱的人
- 候选科学家：
  - `Hedy Lamarr`
  - `Thomas Edison` 也可能兼容

## 6. 关于命名的说明

这里的 16 个名字是**系统工作名**，不是最终一定要给用户看的产品名。

建议把命名分两层：

### 6.1 系统层命名

用于：

- 内部分类
- 科学家标注
- 题目映射
- 画像计算

要求：

- 结构稳定
- 易区分
- 不太花哨

### 6.2 产品层命名

用于：

- 测试结果页
- 分享卡
- 用户认领

要求：

- 更有趣
- 更口语
- 更像“身份标签”

所以现在这 16 个更接近“系统层工作名”，后面完全可以再长出更有趣的产品层别名。

---

## 7. 为什么代表科学家只能先给“候选”

当前不能直接宣布“某类 = 某科学家”，原因有三个：

1. 同一个科学家可能跨多个原型边界
2. 当前 `120` 位人物里，很多仍是 `seeded dossier / seeded skill`
3. 科学家更适合做“锚点候选”，而不是唯一代表

所以现在最合理的规则是：

- 每个原型先配 `2-3` 位候选科学家
- 后续再根据 dossier 补强程度、语料厚度和风格清晰度收缩成：
  - 主锚点
  - 备选锚点

---

## 8. 分类落地规则

后续给每位科学家打标时，建议按这个顺序：

1. 先判四个主轴：
   - O
   - C
   - E
   - S
2. 再判 `I / D`
3. 再给出：
   - 主原型
   - 副原型（可选）
   - 主锚点置信度
   - 代表维度

最终每位科学家的结构建议写成：

```text
主原型：深构建模者
完整编码：OCeS-I
辅助标签：理论 / 抽象建模 / 低调高压缩 / 独立突破
锚点置信度：高
```

## 9. 当前推荐的下一步

现在最值得做的不是继续抽象讨论，而是：

1. 用这 16 个主原型建立正式 rubric
2. 先挑 `20-30` 位强锚点科学家逐个落型
3. 看哪些原型已经有稳定代表人物，哪些还缺
4. 再决定题库和用户测试结果怎么映射回来

一句话说：

**这份文档把分类骨架定住了；下一步该从“定义类型”进入“给科学家逐个打型”。**
